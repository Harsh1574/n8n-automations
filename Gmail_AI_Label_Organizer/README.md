# 🏷️ Gmail Label Organizer — Autonomous AI Email Categorization Agent

> **Built with:** n8n · OpenAI GPT-3.5 · Gmail API  
> **Type:** Event-Driven Automation · AI Agent · Autonomous Tool Use

---

## 🧠 What This Does

This workflow automatically categorizes every incoming email in real-time using an AI Agent. The moment an email arrives, the agent reads it, decides the appropriate label, and applies it to Gmail — all without any human intervention.

Unlike simple rule-based filters, this agent **understands context** — it reads the email body, resolves ambiguity, and even creates new labels autonomously if needed.

---

## 🏗️ Architecture

```
Gmail Trigger (fires on every new email)
    → AI Agent
         ├── Tool: Get all existing Gmail labels
         ├── Tool: Create a new Gmail label (if needed)
         └── Tool: Apply label to email
              ↑
         OpenAI GPT-3.5
```

---

## 🤖 What Makes This an Agent (Not Just a Chain)

Most n8n AI workflows use a **Basic LLM Chain** — you pass data in, get output back. Simple.

This workflow uses an **AI Agent** with tools — meaning the model:
1. **Decides** which tools to call and in what order
2. **Reads** the existing Gmail labels first
3. **Creates** a new label if the category doesn't exist yet
4. **Applies** the correct label using its ID

The agent follows a ReAct pattern — Reasoning + Acting — autonomously deciding its next step based on the results of each tool call.

---

## ⚙️ Key Technical Decisions

### 1. Event-Driven Trigger
Unlike the Daily Email Digest (which runs on a schedule), this workflow uses a **Gmail Trigger** that polls every minute and fires immediately when a new email arrives — ensuring real-time categorization.

### 2. Dynamic Label Creation
The agent doesn't rely on a hardcoded list of labels. If it encounters an email that doesn't fit existing categories, it calls the "Create label" tool and builds the taxonomy autonomously.

### 3. Body Over Subject
Through testing, the agent correctly prioritizes **email body content** over subject lines for categorization — correctly handling:
- Vague subjects with specific bodies
- Misleading subjects that contradict the body
- Mixed language emails (Hindi/English)

### 4. Conflict Resolution via Prompt Engineering
When emails contain keywords from multiple categories (e.g. both "hackathon" and "internship"), the agent applies a **primary intent rule** — identifying what the email is *mainly* about rather than applying multiple labels.

## 📊 Label Taxonomy (Auto-Generated by Agent)

| Label | Description |
|-------|-------------|
| **Hackathons** | Coding competitions, tech events, prize-based contests |
| **Internships** | Job offers, career opportunities, hiring emails |
| **Newsletters** | Subscriptions, promotional updates, digests |
| **Finance** | Transactions, billing, account alerts |
| **Personal** | Emails from real people, friends, colleagues |
| **Daily Digest** | Auto-generated daily email digest from n8n |
| **Other** | Anything that doesn't fit above categories |

> All labels were **created autonomously by the agent** — none were manually created.

## 🧪 Edge Cases Tested

| Test | Subject | Body | Result |

## 🔧 Setup

### Prerequisites
- n8n Cloud account
- Gmail OAuth2 credentials
- OpenAI API key (GPT-3.5-Turbo)

### Configuration
1. Import the workflow JSON into n8n
2. Connect Gmail OAuth2 credential to all Gmail nodes
3. Add OpenAI API key to the Chat Model node
4. Activate the workflow — it runs automatically from here

## 💡 What I Learned

- **AI Agents vs Chains** — agents decide their own tool-calling order at runtime
- **Tool configuration in n8n** — "From AI" mode lets the agent populate parameters dynamically
- **Prompt engineering for agents** — writing explicit step-by-step instructions and conflict resolution rules
- **ReAct pattern** — how agents reason and act iteratively
- **Event-driven vs scheduled triggers** — choosing the right trigger for real-time automation
- **Gmail API label IDs** — agents must use label IDs not names when applying labels

## 🔄 Enhancements

### v1.0 — Initial Build
- Basic AI Agent with 3 Gmail tools
- Agent could categorize emails into predefined categories
- Dynamic label creation if category didn't exist

### v1.1 — Multi-Label Bug Fix
- **Problem:** Agent was applying multiple labels to a single email when the content contained keywords from more than one category (e.g. an email about a hackathon with an internship prize got both "Hackathons" and "Internships" labels)
- **Fix:** Added conflict resolution rules to the prompt instructing the agent to identify the **primary intent** of the email and apply exactly ONE label regardless of secondary mentions
